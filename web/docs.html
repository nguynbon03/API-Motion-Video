Kling Motion Control API

Generate AI motion videos by uploading a character image and a motion reference video. The API handles account rotation, browser automation, and video generation automatically.

REST API Multipart Upload Auto Account Rotation

Quick Start

Create a motion video in 3 steps:

1
Upload image + video
→
2
Poll task status
→
3
Download result
bash
# 1. Create task — upload image + video
curl -X POST https://YOUR_SERVER/api/generate \
  -F "image=@character.png" \
  -F "video=@dance.mp4" \
  -F "mode=pro"
# → {"data": {"task_id": 1, "status": "queued"}}

# 2. Poll status (repeat until succeed)
curl https://YOUR_SERVER/api/tasks/1
# → {"status": "succeed", "result_video_url": "/outputs/task_1.mp4"}

# 3. Download video
curl -O https://YOUR_SERVER/outputs/task_1.mp4

Authentication

Currently no API key required. If using ngrok, include this header:

header
ngrok-skip-browser-warning: true
💡 API key authentication can be added in future versions for production use.

Task Status Flow

Every task goes through these stages:

QUEUED — In queue
SUBMITTED — Account assigned
PROCESSING — Video creating
SUCCEED — Video ready
FAILED — Error occurred
⏱ Typical generation time: 5-10 minutes per video. Poll every 10 seconds.
POST /api/generate Create motion video (all-in-one)

Upload character image and motion video in a single request. Returns a task ID for polling.

Request Body multipart/form-data

FieldTypeDefaultDescription
imageFileREQUIRED—Character image (.jpg, .png). Max 10MB.
videoFileREQUIRED—Motion reference video (.mp4, .mov). 3-30 seconds, max 100MB.
promptstringoptional""Text description of the character.
modestringoptional"pro"pro (best quality) or std (faster).
model_namestringoptional"kling-v2-6"kling-v2-6 or kling-v3
orientationstringoptional"image"image — match image pose. video — match video pose.
keep_soundstringoptional"yes"yes — keep original audio. no — silent.

Response

json — 200 OK
{
  "code": 0,
  "message": "Task queued",
  "data": {
    "task_id": 1,
    "external_task_id": "gen-a1b2c3d4",
    "status": "queued",
    "image": "abc123_character.png",
    "video": "def456_dance.mp4"
  }
}

Example

POST /api/upload/accounts Import Kling accounts

Upload a text file with Kling web accounts. Each line: email:password or email:password:proxy

FieldTypeDescription
fileFileREQUIREDText file with account lines
creditsfloatoptionalDefault credits per account (default: 66)

Account file format

accounts.txt
# Simple format
user1@gmail.com:MyPassword123

# With proxy (for account rotation)
user2@gmail.com:Pass456:socks5://1.2.3.4:1080
user3@gmail.com:Pass789:http://proxy:8080

# Full format: name|email|password|proxy|credits
acc-01|user4@gmail.com|Pass000|socks5://5.6.7.8:1080|100

Code Examples

JavaScript (Browser / Website)

javascript
const API = 'https://YOUR_SERVER';

// 1. Create motion video
async function createVideo(imageFile, videoFile) {
  const form = new FormData();
  form.append('image', imageFile);
  form.append('video', videoFile);
  form.append('mode', 'pro');

  const res = await fetch(`${API}/api/generate`, {
    method: 'POST',
    headers: { 'ngrok-skip-browser-warning': 'true' },
    body: form
  });
  const { data } = await res.json();
  return data.task_id;
}

// 2. Poll until done
async function waitForVideo(taskId) {
  while (true) {
    const res = await fetch(`${API}/api/tasks/${taskId}`, {
      headers: { 'ngrok-skip-browser-warning': 'true' }
    });
    const task = await res.json();

    if (task.status === 'succeed')
      return `${API}${task.result_video_url}`;
    if (task.status === 'failed')
      throw new Error(task.error_message);

    await new Promise(r => setTimeout(r, 10000));
  }
}

Python

python
import httpx, time

API = "https://YOUR_SERVER"

# 1. Create task
files = {
    "image": open("character.png", "rb"),
    "video": open("dance.mp4", "rb"),
}
data = {"mode": "pro", "prompt": "A girl in gray T-shirt"}

r = httpx.post(f"{API}/api/generate", files=files, data=data)
task_id = r.json()["data"]["task_id"]

# 2. Poll
while True:
    r = httpx.get(f"{API}/api/tasks/{task_id}")
    task = r.json()
    if task["status"] == "succeed":
        video_url = f"{API}{task['result_video_url']}"
        break
    time.sleep(10)

# 3. Download
r = httpx.get(video_url)
open("output.mp4", "wb").write(r.content)

Limits & Tips

ParameterLimit
Image sizeMax 10MB, .jpg/.png
Image ratio1:2.5 to 2.5:1
Video sizeMax 100MB, .mp4/.mov
Video duration3-30 seconds (10s max if orientation=image)
Concurrent tasks1 per account (auto-rotation)
Generation time~5-10 minutes per video
Credits per video~19 credits (pro mode, 720p)

Video Tips for Best Results

  • Use video with 1 person clearly visible
  • Head + arms + legs should be visible (minimal occlusion)
  • Avoid fast camera movement or quick cuts
  • Moderate motion speed works best
  • Good lighting on both image and video
